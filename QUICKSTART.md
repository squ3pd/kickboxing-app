# Быстрый старт

## Шаг 1: Создание бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям:
   - Введите имя бота (например: "Кикбоксинг Анализ")
   - Введите username бота (должен заканчиваться на `bot`, например: `kickboxing_analysis_bot`)
4. Сохраните токен бота (выглядит как `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

## Шаг 2: Размещение приложения

### Вариант A: GitHub Pages (бесплатно)

1. Создайте аккаунт на [GitHub](https://github.com) (если нет)
2. Создайте новый репозиторий (например, `kickboxing-telegram-app`)
3. Загрузите файлы:
   - `index.html`
   - `styles.css`
   - `app.js`
4. Перейдите в Settings → Pages
5. Выберите branch `main` и папку `/ (root)`
6. Сохраните
7. Через несколько минут получите URL вида: `https://ваш-username.github.io/имя-репозитория/`

### Вариант B: Vercel (бесплатно, проще)

1. Зайдите на [vercel.com](https://vercel.com)
2. Зарегистрируйтесь через GitHub
3. Нажмите "Add New Project"
4. Импортируйте репозиторий или загрузите файлы
5. Нажмите Deploy
6. Получите URL вида: `https://ваш-проект.vercel.app`

## Шаг 3: Настройка Mini App в BotFather

1. Вернитесь к [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newapp`
3. Выберите созданного бота
4. Введите название приложения (например: "Кикбоксинг")
5. Загрузите иконку (опционально, можно пропустить)
6. Введите URL вашего приложения (из шага 2)
7. Введите краткое описание (опционально)
8. Готово! Получите ссылку на приложение

## Шаг 4: Тестирование

1. Откройте вашего бота в Telegram
2. Отправьте команду `/start`
3. Нажмите на кнопку "Открыть приложение" или используйте меню бота
4. Проверьте работу всех функций:
   - Добавление спортсмена
   - Регистрация тренировки
   - Таблица нормативов

## Шаг 5: Интеграция формул

1. Откройте PDF файл "Шестаков №8.2025.pdf"
2. Найдите математические формулы
3. Откройте файл `app.js`
4. Найдите функции расчета (см. `FORMULAS.md`)
5. Замените формулы согласно инструкции

## Важные замечания

- URL приложения должен быть доступен по HTTPS
- Приложение должно быть доступно публично (не требует авторизации)
- Данные хранятся локально в браузере (LocalStorage)
- Для сохранения данных на сервере потребуется дополнительная разработка

## Решение проблем

### Приложение не открывается
- Проверьте, что URL правильный и доступен
- Убедитесь, что используется HTTPS
- Проверьте консоль браузера на ошибки (F12)

### Данные не сохраняются
- Проверьте, что LocalStorage доступен в браузере
- Очистите кэш браузера и попробуйте снова

### Формулы работают неправильно
- Проверьте правильность формул в `app.js`
- Убедитесь, что все параметры передаются корректно
- Проверьте консоль браузера на ошибки
